The Machine Learning Control framework contains two Github actions that automatically check and deploy new documentation:

  • The docs_check_ci action checks your changes to see if the documentation still builds.
  • The docs_publish_ci action deploys your documentation if a new version of the mlc framework is released.

Automatic build instructions¶

To successfully deploy your new documentation, you have to follow the following development steps:

  1. Create a new branch on for the changes you want to make to the documentation (e.g. docs_change branch).
  2. Make your changes on this branch.
  3. Commit your changes. This will trigger the docs_check_ci action to run.
  4. Create a pull request into the main branch if this action ran without errors.
  5. Add a version bump label bump:patch, bump:minor or bump:major to the pull request.
  6. Merge the pull request into the main branch. The documentation will now be deployed using the docs_publish_ci action.

Manual build instructions¶

Install requirements¶

Building the Machine Learning Control’s HTML documentation requires sphinx, the Machine Learning Control package and several plugins. All of the above can be installed using the following pip command:

pip install -e .[docs]

If you also want to build the LATEX documentation, you have to install the texlive-full package.

Build the documentation¶

Build HTML documentation¶

To build the HTML documentation go into the docs/ directory and run the make html command. This command will generate the html documentation inside the docs/build/html directory.

Note

Make sure you are in the Conda environment in which you installed the Machine Learning Control package with it’s dependencies.

Build LATEX documentation¶

To build the LATEX documentation go into the docs/ directory and run the make latex command. This command will generate the html documentation inside the docs/build/latex directory.

Deploying¶

To deploy documentation to the Github Pages site for the repository, push the documentation to the main branch and run the make gh-pages command inside the docs/ directory.

Warning

Please make sure you are on the main branch while building the documentation. Otherwise, you will be greeted by errors.
