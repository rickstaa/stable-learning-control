Code formatting guidelines¶

To increase code quality, readability and usability, we use several language guidelines:

Python guidelines:
  • Linting: Please make sure your python code doesn’t contain any errors by checking it with the flake8 python linter.
  • Formatting: Please format all your scripts using the black python formatter.
Markdown guidelines:
  • Linting and formatting: Please make sure your markdown code contains no errors and is formatter according to the remark-lint style guidelines.

The Machine Learning Control package contains several GitHub actions, which check code changes against these coding guidelines. As a result, when the above guidelines are not met, you will receive an error/warning when you create a pull request. If you think a code guideline is not correct or your code structure doesn’t allow you to respect the guideline, please state so in the pull request.

General guidelines¶

Before releasing the package, make sure the following steps are performed:

  1. Create a pull request to pull the changes of your development branch onto the master branch.
  2. Make sure that all the pull request checks were successful.
  3. Squash and merge your branch with the main branch.
  4. Update the documentation according to Release documentation if needed.
  5. Bump the version using the bump2version tool.
  6. Check the version of the current branch using the bumpversion --list command.
  7. Add a tag equal to the version specified in the last step (Check versioning guidelines below).
  8. Update the changelog using the auto-changelog tool.
  9. Commit and push the changes to the remote.
  10. Create a release using the GitHub draft release tool.

Versioning guidelines¶

Additionally please use the versioning guidelines specified at semver.org.
